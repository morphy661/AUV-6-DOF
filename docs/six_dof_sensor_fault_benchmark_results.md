# 六自由度传感器故障开发基准结果

## 基准性质

本结果是开发阶段评估，不是冻结盲测，也不能解释为真实海试准确率。它用于验证深度、IMU、DVL规则诊断、状态估计回退与FTC动作之间的接口是否正确，并暴露当前传感器组合的可观测性边界。

评估日期为2026-07-17。基准包括10类场景，每类5个噪声与扰动种子，共50个任务：

- 正常任务5个；
- 深度、IMU、DVL分别注入不可用、卡死、突变，每类5个；
- 每个任务18 s，步长0.05 s；
- 5–9 s为故障区间，任务包含多段位置、深度和偏航机动；
- 深度、IMU、DVL噪声以及小幅六轴环境扰动按任务随机化；
- 控制器使用上一时刻传感器估计，仿真真值只用于评价。

运行命令：

```powershell
python math-model/examples/evaluate_six_dof_sensor_faults.py --missions-per-scenario 5 --strict
```

## 发现并修复的问题

第一次20任务冒烟评估的事件精确率只有56.25%。检查发现两类恢复逻辑问题：

1. 不可用或卡死结束后的首个正常读数，与旧读数差异较大，被重复报告为突变；
2. DVL卡死已经确认后，短暂缺少运动激励会把同一事件拆成多个片段。

修复后，首个恢复样本只用于重新建立基线；确认卡死保持锁存，直到读数真正变化。相同20任务复测和最终50任务均没有附加事件。

## 绝对位置不可恢复的原因

深度、IMU和DVL可以提供深度、姿态、角速度、加速度和机体系速度，但不提供绝对水平位置。持续IMU/DVL故障产生的水平航位推算偏差，在传感器恢复后仍缺少观测量进行校正。

因此当前输出必须区分：

- `传感器已恢复`：当前数据重新有效；
- `水平位置参考仍不确定`：累计位置偏差没有外部修正；
- `FTC保持降级导航`：不能因为传感器重新输出就宣称任务完全恢复。

状态估计器已经提供外部水平位置修正接口。未来接入USBL/LBL、声学定位、周期性浮出GPS或其他绝对位置源后，才能重新建立水平位置参考并恢复正常航行。

## 对最终演示文字的影响

在当前开发条件下，明显不可用、全通道卡死和超过物理阈值的突变可以显示为“确定的传感器故障”。但应同时显示位置完整性：

- `确定：IMU卡死；安全保持；水平位置参考降级`；
- `确定：DVL不可用；惯性航位推算；等待外部位置修正`；
- `确定：深度突变；当前样本已拒绝；未触发推进器隔离`。

弱推力损失和具体推进器位置仍使用“可能故障＋组别/Top-2概率”，不能因为本传感器开发基准通过而改成确定定位。

## 下一验证门槛

在正式声称高准确率前，需要冻结新种子并增加更困难的独立任务：弱突变、部分通道卡死、间歇失锁、缓慢偏置/漂移、故障与强环境扰动重叠，以及外部位置修正可用/不可用两种条件。当前50任务结果不得用于反复调阈值后再作为盲测结果。
