# 六自由度传感器故障冻结压力测试 V1

## 结论

冻结 V1 完成了 25 类场景、每类 3 个全新种子，共 75 次 18 秒闭环任务。控制器只使用因果传感器估计，注入真值只进入离线评分器。

强直接故障层通过全部安全与准确性指标，但模糊故障层没有通过预设覆盖门槛。因此，当前系统可以可靠处理明显故障，却还不能把偏置、漂移、弱异常和局部通道故障统一转化为“可能故障”提示。

| 指标 | 冻结 V1 结果 |
|---|---:|
| 强直接故障事件召回率 | **100%** |
| 强直接故障事件精确率 | **100%** |
| 强直接故障 FTC 动作匹配率 | **100%** |
| 正常任务误确认 | **0 / 6** |
| 正常任务错误保护动作 | **0 / 6** |
| 模糊故障任意证据覆盖率 | **35.71%** |
| 模糊故障“仅可能”提示率 | **0%** |
| 模糊故障精确分类率 | **21.43%** |
| 模糊故障错误确定率 | **14.29%** |
| 恢复后残留保护动作 | **0** |
| 传感器故障错误指定推进器 | **0** |

总体 `all_acceptance_checks_passed=false`。失败项是模糊证据覆盖率低于预设 50%，以及错误确定率高于预设 10%；强直接诊断、正常工况安全性和 FTC 隔离边界均通过。

## “确定”和“可能”的展示边界

| 观测情况 | 当前展示等级 | 原因 |
|---|---|---|
| 明显不可用，数据无效或非有限值 | `确定：当前传感器数据不可用` | 三种传感器冻结任务均 3/3 正确 |
| 全通道持续不变，且独立信号要求运动 | `确定：传感器卡死` | 三种传感器冻结任务均 3/3 正确，先经过确认时间 |
| 超过物理跳变阈值的强尖峰 | `确定：异常采样，已拒绝本次数据` | 三种传感器冻结任务均 3/3 正确 |
| 弱尖峰 | `记录：短时异常；可能存在传感器问题` | 0/9 任务形成可用证据，不得写确定 |
| 小幅恒定偏置 | `可能：传感器偏置；继续观察` | 0/9 任务形成可用证据 |
| 缓慢漂移 | `可能：传感器漂移；继续观察` | 0/9 任务形成可用证据 |
| IMU/DVL 单通道局部卡死 | `可能：局部通道异常` | 能观察到异常，但恢复跳变被错误确认为尖峰，不能写确定 |
| 短时反复掉线 | `确定：当前样本不可用` + `可能：间歇性硬件/通信问题` | 每次无效样本是直接事实；硬件永久损坏并未被证明 |

传感器异常不得用于指定某台推进器。传感器 FTC 只能触发样本拒绝、降级导航或安全保持；推进器隔离仍必须依赖推进器本地电流/RPM等独立证据。

## 各模糊场景结果

| 场景组 | 任意证据 | 精确分类 | 错误确定 | 当前解释 |
|---|---:|---:|---:|---|
| 深度/IMU/DVL 弱尖峰 | 0% | 0% | 0% | 未检测，保留日志级 |
| 深度/IMU/DVL 偏置 | 0% | 0% | 0% | 需要长窗残差层 |
| 深度/IMU/DVL 漂移 | 0% | 0% | 0% | 需要趋势和跨传感器一致性层 |
| IMU/DVL 局部通道卡死 | 100% | 0% | 100% | 恢复跳变被误认为尖峰，应优先修复 |
| 深度/IMU/DVL 间歇不可用 | 100% | 100%（样本状态） | 0% | 即时 FTC 正确，维护结论必须降级为可能 |

这里的“错误确定”指在真实局部卡死窗口附近，规则层确认了不同类型的故障；它不代表把正常任务误报为故障。正常和强扰动任务的误报仍为零。

## 下一版最小改动

下一步不需要重训 BiLSTM。先在规则诊断与最终展示之间增加轻量的“长期观察层”：

1. 对 IMU 和 DVL 分通道保存基线，避免单通道恢复跳变被确认为新的尖峰；
2. 用 2–5 秒滑动窗口记录小幅残差的均值和斜率，输出 `possible_bias` 或 `possible_drift`，只记日志、不直接触发推进器 FTC；
3. 对短时反复不可用分开保存“样本不可用事实”和“间歇性硬件问题可能性”；
4. 保持现有强故障快速安全路径不变；
5. 完成后用新的 V2 哈希、新种子和新目录验证，不能覆盖本次 V1。
