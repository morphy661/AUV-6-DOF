# 六自由度传感器故障、FTC 与视频回接路线

## 当前已完成

本阶段没有立即扩大 BiLSTM-Attention 的类别数，而是先实现能由机载信号直接确认的传感器故障。这样可避免把规则可以可靠判断的问题变成更难验证的学习问题。

统一接口覆盖三个传感器和三类故障：

| 传感器 | 不可用 | 卡死 | 突变 |
|---|---|---|---|
| 深度 | 非有限值或无效标志 | 有垂向运动证据但读数持续不变 | 单步深度跳变超过物理阈值 |
| IMU | 姿态、角速度或加速度数据无效 | 有转动指令但整组读数持续不变 | 姿态、角速度或加速度单步跳变 |
| DVL | 速度无效或失锁 | 有加速度/速度指令变化但读数持续不变 | 任一速度轴单步跳变 |

卡死检测必须同时存在独立的“应该发生运动”证据，所以 AUV 正常定点悬停时连续相同读数不会自动被判为故障。

每个仿真步输出：

- `sensor_health.depth/imu/dvl`：故障类型、置信度、可信等级、是否确认和建议动作；
- `sensor_health_summary`：不可用传感器集合与 FTC 建议；
- `ftc_sensor_guard_action`：FTC 实际接收的传感器保护建议；
- `ftc_untrusted_sensors`：本步被排除的传感器；
- 原有位置、姿态、六台推进器遥测、故障概率和 FTC 动作。

注入真值 `sensor_fault_truth` 只用于仿真评价。在线诊断和 FTC 证据构造器不读取该字段，防止标签泄漏。

## FTC 动作边界

| 诊断结果 | 当前 FTC 行为 |
|---|---|
| 单点突变 | 拒绝当前样本并记录，不隔离推进器 |
| 尚未确认的卡死 | 继续观察，不隔离推进器 |
| 深度或 DVL 确认不可用/卡死 | 降低导航需求，准备使用剩余估计来源 |
| IMU 确认不可用/卡死 | 请求安全保持或终止任务，不猜测推进器位置 |
| ESC 电流和 RPM 直接确认某台推进器无输出 | 定向隔离并在下一步重新分配推力 |

直接 ESC 无输出证据仍优先执行一次确定性隔离。传感器故障、神经网络 Top-1/Top-2 或弱推力损失概率都不能单独关闭某台推进器。

## 后续最简路线

### 1. 状态估计与回退（已完成轻量版本）

六自由度仿真器现在默认采用“本步传感器更新估计，下一控制步使用估计”的因果结构。仿真真值只保留在评价字段；设置 `use_sensor_feedback=False` 才会显式回到真值控制基准。当前轻量估计层已经实现：

- 深度失效：短时间使用惯导/DVL 垂向速度外推，并限制深度机动；
- DVL 失效：使用 IMU 短时航位推算，同时增加速度不确定度；
- 单点突变：拒绝该样本并使用上一时刻估计；
- IMU 失效：不继续激进姿态控制，进入安全保持或任务终止流程。

估计日志同时保存控制器实际使用的状态、当前估计状态、估计来源、被排除/拒绝的传感器和回退持续时间，可以直接供最终视频显示。

该版本用于建立正确闭环边界，不等同于生产级 INS/EKF：水平位置依靠 DVL/IMU 航位推算，长时间失锁仍会漂移；IMU 加速度目前使用仿真中的机体系加速度接口，尚未加入真实 IMU 的重力补偿、偏置估计和协方差传播。这些内容在独立任务验证后按需要增加，不在当前阶段训练新的大模型。

### 2. 独立传感器故障任务

为深度、IMU、DVL 分别生成正常、不可用、卡死、突变任务，并加入海流、噪声、参数随机化和不同机动强度。评价重点不是单一窗口准确率，而是：

- 故障事件召回率和确认延迟；
- 正常任务误保护次数；
- 短时异常恢复后是否自动返回正常；
- FTC 是否采取了与传感器类型匹配的动作；
- 是否出现错误推进器隔离。

只有规则无法可靠覆盖的模糊故障，再考虑加入 BiLSTM-Attention 的独立传感器辅助头。

### 3. 六自由度实时回放与 MP4

最终视频从同一份仿真日志逐帧渲染，不使用未来信息，建议布局为：

1. 左侧：AUV 三维轨迹、当前位置、目标点和滚转/俯仰/偏航；
2. 右上：深度、IMU、DVL 的实时健康灯、故障类型与置信度；
3. 右中：H1-H4、V1-V2 的指令、电流/RPM 和故障位置概率；
4. 右下：FTC 当前动作、目标推进器、推力重分配和事件时间线。

输出包括 MP4、逐步 CSV/JSON 日志和一张静态总结图。现有 `math-model/src/utils/visualization.py` 已具备 Matplotlib `FuncAnimation` 与 FFmpeg 保存能力，后续增加专用六自由度诊断渲染器即可。

## 当前验证

新增诊断、传感器注入、FTC 优先级、状态估计闭环、开发基准和防真值泄漏测试后，`math-model/tests` 完整回归为 **136/136 通过**。零噪声 20 秒闭环稳定性检查的最大位置估计误差约为 **0.023 m**，最终目标误差约为 **0.169 m**；该结果只验证确定性接口和连续运行稳定性，不代表真实海试精度。

独立传感器开发矩阵包含50个任务，事件Precision/Recall、FTC动作匹配率和传感器健康恢复率均为100%，正常任务误保护和错误推进器定位均为0。绝对轨迹恢复率为73.33%，明确暴露了持续IMU/DVL故障后缺少绝对水平位置修正的问题。完整说明见 `docs/six_dof_sensor_fault_benchmark_results.md`。
