## 最终展示政策

### 1. 确定

以下直接可观测故障可以显示 `确定`，并进入原有传感器安全路径：

- 深度、IMU、DVL 当前数据明确不可用；
- 全通道持续卡死，并有独立运动证据；
- 超过物理跳变阈值的强尖峰。

对应 FTC 只允许样本拒绝、降级导航或安全保持，不能据此指定推进器。

### 2. 可能

以下场景必须显示 `可能`，只记录和观察，不触发 FTC：

- IMU 或 DVL 局部通道卡死；
- 深度、IMU 或 DVL 持续偏置/一致性残差；
- IMU 慢漂移；
- 深度、IMU 或 DVL 在短时间内反复不可用。

间歇不可用需要同时显示两个层次：`当前样本确定不可用`，但 `硬件或通信根因可能异常`。

### 3. 可选可能

深度和 DVL 慢漂移不强求全部识别：

- 长期残差证据充分时，允许显示 `可能：偏置或漂移`；
- 证据不足时只保留后台日志。

V3 中深度慢漂移 1/3、DVL 慢漂移 1/3 显示为可能，其余保持日志。该差异不会触发错误维护或 FTC。

### 4. 必须仅日志

深度、IMU、DVL 的单次弱尖峰只作为背景事件保存，不产生操作者维护提示。V3 的 9 个任务全部保持日志级。

## 关于兼容字段

V3 JSON 仍保留一个来自 V2 汇总器的兼容字段 `log_only_policy_overpromoted_missions=2`，它按照旧 V2 政策把两个可选慢漂移提示计为升级。V3 的正式验收字段是 `required_log_only_overpromoted_missions=0`，仅统计三类单次弱尖峰。`all_acceptance_checks_passed=true` 使用的是 V3 冻结政策。
