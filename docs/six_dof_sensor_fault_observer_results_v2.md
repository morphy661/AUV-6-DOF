# 六自由度传感器长期观察层冻结 V2 结果

## 本阶段完成的修改

V1 压力测试表明原规则层几乎只有“确定/正常”，缺少可靠的“可能”中间层。V2 在不修改 BiLSTM、推进器诊断和 FTC 决策规则的前提下增加了因果长期观察层：

- 弱跳变只写入后台日志；
- 使用 5 秒传感器/运动学一致性残差记录可能偏置或漂移；
- 分通道检测 IMU/DVL 局部卡死；
- 局部卡死恢复样本用于重建基线，不再被错误确认为尖峰；
- 同一传感器在 10 秒内反复不可用时，记录可能的间歇硬件或通信问题；
- 观察层固定 `confirmed=false`、`ftc_recommendation=none`，不能指定推进器。

完整回归测试为 **153/153 通过**。

## 唯一失败项的含义

原 V2 协议把深度和 DVL 慢漂移都强制规定为“只允许日志”。实际结果中：

- 3 个深度慢漂移任务有 1 个形成了持续一致性残差，并正确输出 `可能：偏置或漂移`；
- 另外 2 个深度慢漂移任务仍为日志；
- 3 个 DVL 慢漂移任务均未升级为操作者提示；
- 没有正常任务被升级，没有传感器或故障类型冲突，也没有 FTC 动作。

因此这一次失败不是误报、错误类型或安全越权，而是冻结展示政策过度限制了真实故障证据。V2 结果仍必须按“未完全通过”保存，不能在看到结果后修改原协议并重新解释为通过。

## 当前可采用的展示边界

| 场景 | 展示政策 |
|---|---|
| 强不可用、全通道卡死、强尖峰 | `确定`，进入现有快速传感器安全路径 |
| IMU/DVL 局部通道卡死 | `可能`，记录通道，不直接触发 FTC |
| 深度/IMU/DVL 持续偏置证据 | `可能：偏置或漂移` |
| IMU 慢漂移 | `可能：偏置或漂移` |
| 反复短时不可用 | 当前样本 `确定不可用`；硬件/通信根因 `可能` |
| 单次弱尖峰 | 后台日志，不打扰操作者 |
| 深度/DVL 慢漂移 | 可选 `可能`：有持续残差时允许提示，没有时保留日志；不能强求全部检出 |

## 下一步

不需要继续降低检测阈值，也不需要重训 BiLSTM。下一步应只修正冻结展示政策：

1. 保留 9 个“必须显示可能”的场景；
2. 把深度/DVL慢漂移从“禁止升级”改成“证据充分时允许可能，否则日志”；
3. 只把三类单次弱尖峰保留为强制日志级；
4. 使用新协议、新哈希、新种子执行 V3，不覆盖 V2。
